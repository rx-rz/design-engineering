01 / The Concept

Every element is a box.

The browser wraps every HTML element in four nested regions. From inside out: content ‚Üí padding ‚Üí border ‚Üí margin. Understanding these four zones is the foundation of all CSS layout.

02 / box-sizing: border-box

The fix that should be the default.

By default, box-sizing: content-box means width only sets the content area. Padding and border get added on top. That's unintuitive and breaks layouts. border-box makes width include padding and border ‚Äî what you set is what you get.

03 / The Weird Part

Margin Collapsing

When two vertical margins of block elements meet, they don't add ‚Äî they collapse into the larger of the two. This only happens vertically (top/bottom), never horizontally (left/right).

What you might expect
Block A (margin-bottom: 32px)
‚Üì 32px
‚Üë 32px
Block B (margin-top: 32px)

‚ùå You'd think: 32 + 32 = 64px gap

What actually happens
Block A (margin-bottom: 32px)
‚Üê collapsed to 32px (the larger value)
Block B (margin-top: 32px)

‚úì Reality: max(32, 32) = 32px gap

The rules:

  • WHEN Adjacent siblings ‚Äî margin-bottom of one + margin-top of next ‚Üí collapses to the larger value.
  • WHEN Parent + first/last child ‚Äî if no border/padding/BFC separates them, the child's margin merges with the parent's.
  • NEVER Horizontal margins never collapse. Flexbox/grid children never collapse. Elements with overflow ‚â† visible, display: flex, or position: absolute don't collapse.
  • FIX Add padding: 1px, border, overflow: hidden, or make the parent a flex/grid container to prevent unwanted collapse. 
/* Classic parent-child collapse trap */
.parent {
  background: blue;
  /* No padding, no border ‚Üí child's margin-top "escapes" */
}
.child {
  margin-top: 40px; /* this pushes the PARENT down, not the child inside it */
}

/* Fix: give parent a border or padding-top */
.parent {
  padding-top: 1px; /* creates a barrier, stops collapse */
}

05 / Key Points

Remember These

Box Model

Content ‚Üí padding ‚Üí border ‚Üí margin. Every element. Every time. The browser calculates layout from these four values.

border-box

Always use *, *::before, *::after { box-sizing: border-box }. Width and height include padding and border. No surprises.

Margin Collapsing

Vertical margins between block siblings collapse to the max, not the sum. Only vertical. Not in flex/grid. Parent + child can merge too.

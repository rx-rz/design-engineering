01 / The Mental Model

Two axes, two roles

Flexbox is one-dimensional: it works along a single main axis. The flex-direction sets which axis is main. Items are distributed along the main axis; they're aligned along the cross axis.

02 / Properties

Container props vs item props

The most common flexbox mistake: applying item properties to the container, or vice versa. Know which is which.

On the container (display: flex)
flex-direction
row | column | row-reverse | column-reverse. Sets main axis. flex-row flex-col
flex-wrap
nowrap (default) | wrap | wrap-reverse. Do items wrap to next line? flex-wrap
justify-content
Distribution along the main axis. justify-start justify-between
align-items
Alignment on the cross axis for all items. items-center items-stretch
align-content
Distributes rows when wrapping. Only works with flex-wrap. content-start
gap
Space between items. Replaces margin hacks. gap-4
On each item (the children)
flex-grow
How much the item grows to fill extra space. 0 = don't grow. grow grow-0
flex-shrink
How much the item shrinks when space is tight. 0 = never shrink. shrink-0
flex-basis
Starting size before grow/shrink applies. Like width on the main axis. basis-1/2
flex
Shorthand: grow shrink basis. flex: 1 = 1 1 0. flex-1 flex-auto
align-self
Override align-items for this one item only. self-end self-center
order
Change visual order without changing DOM order. Default 0. order-first
03 / The Trinity

flex-grow, flex-shrink, flex-basis

These three properties control how items respond to available space. Understand them individually first — then flex: 1 will make total sense.

06 / Key Points

Flexbox fundamentals.

flex: 1 = 1 1 0%

flex-basis 0% means items start from nothing and share space equally. flex: auto starts from content size — very different results. flex-1

shrink: 0 for fixed items

Add flex-shrink: 0 to icons, avatars, and fixed-size elements so they never get squished when siblings need space. shrink-0

justify vs align

justify-content → main axis. align-items → cross axis. These swap when you change flex-direction. Memorize the axes, not the properties.

gap not margin

Use gap on the flex container instead of adding margin to every child. Cleaner, no first/last child edge cases. gap-4 gap-x-2

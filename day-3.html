04 / Normal Flow

What is "normal flow"?

Normal flow is the browser's default layout algorithm — no flexbox, no grid, no float, no absolute positioning. Elements arrange themselves following two rules: block elements stack vertically in a block formatting context (BFC), inline elements lay out horizontally in an inline formatting context (IFC).

You break out of normal flow with position: absolute, position: fixed, float, and by creating flex/grid containers. Understanding flow is essential for knowing why these tools behave the way they do. 

/* Default display values the browser applies (you can always override) */
div, section, article, p, h1, ul, li  → block
span, a, strong, em, label            → inline
button, input, select, textarea       → inline-block  /* roughly */
head, script, style                   → none           /* not rendered */
05 / The Hidden Stuff

Anonymous boxes

The browser silently wraps bare text nodes in anonymous boxes when it needs to mix block and inline content. You can't style them directly — you'll never see them in DevTools by name — but they affect layout in surprising ways.

Your HTML:

<div>
  <p>A paragraph</p>
  Some loose text ← no tag, just text
  <p>Another paragraph</p>
</div>
What the browser actually creates:

The browser wraps the orphaned text in an anonymous block box so it can sit alongside the <p> elements in a block formatting context. Same happens in reverse — if you put a block element inside an inline, the browser generates anonymous block wrappers to fix the invalid nesting.

06 / Display Deep Dive

5 things about display to internalize

01

display sets two things

The outer type (how it participates in flow: block or inline) and the inner type (how its children are laid out). display: flex means outer = block, inner = flex. display: inline-flex means outer = inline, inner = flex.

02

Inline top/bottom margin is a lie

margin-top and margin-bottom on inline elements are completely ignored. Only margin-left and margin-right work. Use inline-block if you need vertical spacing.

03

display: none vs visibility: hidden

none removes the element from flow entirely — no space reserved. visibility: hidden hides it but keeps its space. Use the right one or your layout will jump.

04

Replaced elements are weird

img, video, iframe are technically inline but behave like inline-block — they accept width/height. The browser replaces their content with an external resource, hence "replaced."

05

Absolute/fixed breaks out

Setting position: absolute or position: fixed makes an element generate a block box regardless of its display value. An inline span becomes block-like once positioned. 

/* display: flex — outer is block, inner is flex */
.container { display: flex; }          /* starts on new line, children use flex */
.container { display: inline-flex; }   /* flows inline,  children use flex */

/* removed vs hidden */
.gone   { display: none; }            /* element + its space = gone */
.hidden { visibility: hidden; }       /* invisible but space remains */

/* opacity: 0 — also invisible, space remains, but events still fire! */
.ghost  { opacity: 0; }               /* still clickable! */
07 / Key Points

Take these away.

Block block

Full width, stacks vertically, respects all margin/padding. Default for structural elements: div, p, section.

Inline inline

Flows with text, width = content only, ignores top/bottom margin. Don't try to set width/height on a pure inline — it won't work.

Inline-block inline-block

Best of both: flows with text and accepts width, height, and all margin/padding. The classic way to make horizontally laid-out boxes (before flex).

Normal Flow

The default. Blocks stack. Inlines flow. Anonymous boxes fill gaps. Everything else — flex, grid, position — is an intentional escape from this baseline.

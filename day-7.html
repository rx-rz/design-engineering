01 / Color Spaces

hsl vs oklch — perceptual uniformity wins

All color spaces can represent the same colors. The difference is in how they're structured. oklch is perceptually uniform — which means equal steps in lightness/chroma look equal to your eye. hsl is not.

hex / rgb
legacy · device-oriented
Machine-readable but unintuitive. Hard to manipulate meaningfully. no — learn to read, but don't design in hex.
hsl
hue · saturation · lightness
Human-readable. Great improvement over hex. But "50% lightness" looks wildly different between hues — yellow at 50% L is much brighter than blue. not uniform
oklch
lightness · chroma · hue
Perceptually uniform — the same L value looks equally bright across all hues. Easy to create harmonious palettes by varying H while keeping L and C fixed. use this
02 / oklch Anatomy

Three values, total control

oklch(60% 0.18 250)
L — Lightness
60%
0% = black, 100% = white. Perceptually uniform — 60% looks 60% bright regardless of hue. Use this to build consistent tonal scales.
C — Chroma
0.18
Color intensity. 0 = grey, ~0.4 = maximum vivid (varies by hue/L). Unlike HSL saturation, chroma is an absolute value — not percentage of possible.
H — Hue
250
0–360° color wheel. Blues ~250°, Greens ~140°, Reds ~30°, Purples ~310°. Rotate this to get analogous colors; add 180° for complement.
Build an oklch color oklch(60% 0.18 250)

/* oklch palette — consistent brightness across hues */
:root {
  /* Keep L and C fixed, rotate H → harmonious palette */
  --blue:   oklch(60% 0.18 250);
  --green:  oklch(60% 0.18 140);
  --red:    oklch(60% 0.18 30);
  --purple: oklch(60% 0.18 310);

  /* Tonal scale — fix H and C, vary L */
  --blue-100: oklch(95% 0.04 250);
  --blue-300: oklch(75% 0.12 250);
  --blue-500: oklch(60% 0.18 250);
  --blue-700: oklch(40% 0.18 250);
  --blue-900: oklch(20% 0.10 250);
}

03 / color-mix()

Native CSS color mixing

No more JavaScript or preprocessors to blend colors. color-mix() mixes two colors in any color space. Mixing in oklch produces perceptually smooth results — unlike mixing in srgb which can create muddy mid-tones.

color-mix(in oklch, oklch(60% 0.18 250) 50%, white)

color-mix(in oklch, oklch(60% 0.18 250) 50%, black)

/* Tints & shades from a single variable */
.button { background: var(--brand); }
.button:hover { background: color-mix(in oklch, var(--brand) 85%, black); }
.button:active { background: color-mix(in oklch, var(--brand) 70%, black); }

/* Transparent variant */
.tooltip { background: color-mix(in oklch, var(--brand) 15%, transparent); }

04 / CSS Custom Properties

Variables — deeper than you think

CSS custom properties aren't just variables — they cascade, inherit, and can be set at different scopes. They're the foundation of component theming, dark mode, and design token systems.

/* Declaration: always on an element (scope matters) */
:root  { --color: oklch(60% 0.18 250); }  /* global */
.card  { --color: oklch(60% 0.18 30);  }  /* scoped to .card */
[data-theme="dark"] { --bg: #111; --text: #eee; }

/* Usage with fallback */
.button { background: var(--color, oklch(60% 0.18 250)); }

/* Custom properties accept ANY valid token — even partials */
:root { --spacing: 8px; }
.box  { padding: calc(var(--spacing) * 2); }  /* 16px */

/* They can hold non-CSS values too — read by JS */
:root { --brand-hue: 250; }
.el   { color: oklch(60% 0.18 var(--brand-hue)); }  /* hue from variable! */
They cascade. A variable defined closer to an element wins — just like regular CSS specificity.
They inherit. Children see the parent's variables unless overridden. Perfect for component theming: set variables on the component root, consume inside.
They can hold anything. Not just colors — spacing values, font stacks, timing functions, partial values used in calc(). Even raw numbers.
JavaScript can read and write them. el.style.setProperty('--hue', 250). This is how runtime theming works.

05 / Key Points

Color essentials.

Use oklch

Perceptually uniform — equal lightness steps look equal. Build palettes by keeping L and C fixed, rotating H. Tailwind v4 uses oklch natively. text-blue-500

Custom property scope

Define on :root for globals. Override on component roots for theming. Use var(--x, fallback) for safe consumption. They cascade and inherit.

color-mix()

Native tints and shades. Always mix in oklch for clean results. Replaces Sass lighten()/darken() with no preprocessor needed.

Gradients in oklch

CSS gradients can interpolate in oklch: linear-gradient(in oklch, blue, red). Avoids the muddy grey mid-point you get with sRGB interpolation.

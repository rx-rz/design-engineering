02 / The Unitless Rule

line-height ‚Äî use unitless values

The most important thing about line-height: always use unitless numbers, not px or em. A unitless value is a multiplier of the element's own font-size ‚Äî and it inherits that way too, preventing a subtle but nasty bug.

‚ùå line-height: 24px px (bad)
Normal text at 16px ‚Äî 24px line height looks fine here. But the child below inherits 24px literally...
Big heading ‚Üë Inherits 24px ‚Äî way too tight for 32px text üò¨
‚úì line-height: 1.5 unitless (good)
Normal text at 16px ‚Äî 1.5 √ó 16 = 24px. Perfect. The child below inherits the multiplier, not the pixel value...
Big heading ‚Üë Inherits 1.5 ‚Äî now 1.5 √ó 32px = 48px. Proportional ‚úì

/* ‚úó px ‚Äî inherits as a fixed pixel value */
body { line-height: 24px; }  /* children inherit 24px ‚Äî wrong for headings */

/* ‚úó em ‚Äî computed value (px) is inherited, not the em multiplier */

/* ‚úì unitless ‚Äî the multiplier itself is inherited */
body { line-height: 1.6; }    /* children inherit 1.6, calc against their own font-size */

/* Good defaults by use case */

h1    { line-height: 1.1; }   /* headings ‚Äî tight */
.caption { line-height: 1.4; } /* small text ‚Äî slightly looser */

06 / Loading Fonts Right

Google Fonts & @font-face best practices

DO: Use display=swap in your Google Fonts URL ‚Äî shows fallback text while the font loads, avoiding invisible text flash.
DON'T: Load more weights than you use. Every weight is a separate file. Loading 9 weights √ó 2 styles = 18 requests. Pick 2‚Äì3 weights max.
DO: Subset with &text=ABC... for display fonts used sparingly ‚Äî loads only the glyphs you need. Massive savings for icon or logo fonts.
PERF: Self-hosting via @font-face is often faster than CDN ‚Äî no DNS lookup, better caching control. Tools like fontsource (npm) make this easy.
DO: Always specify a system font fallback. 'Playfair Display', Georgia, serif ‚Äî if the web font fails, users still get a reasonable experience.
MODERN: Use font-display: optional for fonts that aren't critical ‚Äî browser uses cached version or skips entirely. Zero layout shift. 
/* Google Fonts ‚Äî efficient URL */
/* Only 2 weights, display=swap, preconnect for speed */
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:wght@400;700&display=swap" rel="stylesheet">

/* Self-hosted @font-face */
@font-face {
  font-family: 'MyFont';
  src:
    url('/fonts/myfont.woff2') format('woff2'),  /* modern */
    url('/fonts/myfont.woff')  format('woff');   /* fallback */
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}

/* font-display values */
/* auto    ‚Äî browser default */
/* block   ‚Äî invisible text briefly, then swap */
/* swap    ‚Äî show fallback immediately, swap when loaded */
/* fallback ‚Äî short block, then fallback, no late swap */
/* optional ‚Äî use font only if cached, else skip */
07 / Key Points

Typography rules to live by.

Unitless line-height

Always. A number like 1.6 is a multiplier that inherits correctly. px and em both inherit the computed pixel value ‚Äî the wrong behavior for headings.

text-wrap: balance

One line on headings, cards, captions. The browser does the math. Don't use it on long body text ‚Äî it's expensive to compute for many characters.

Variable fonts

One file, continuous weight/width/etc axes. Better performance than multiple files. Enables hover animations on font-weight that static fonts can't do.

font-display: swap

Always set this on @font-face. Shows fallback text while loading, preventing the flash of invisible text (FOIT) that makes pages look broken.
